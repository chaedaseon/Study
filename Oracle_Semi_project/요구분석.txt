2. 모든 기능을 로그인을 거친 후에 사용
-> ID, PW 입력(INSERT) -> 로그인(확인) 클릭(SELECT)

3. 관리자는 여러 명의 교수자를 사전에 등록
-> 관리자가 교수 정보 등록(주민번호=패스워드) -> 교수테이블에 INSERT 

교수자가 로그인 과정을 통해 자신이 강의한 과목에 성적 처리 업무
-> 교수자가 성적 등록(INSERT) -> PROCEDURE

관리자는 등록된 모든 교수자의 정보 출력
<출력 정보> 교수자명, 배정된 과목명, 과목기간, 교재명, 강의실, 강의진행여부 -> VIEW

4. 최종 관리자는 여러 개의 과정을 미리 등록(한 개의 과정은 여러 개의 과목으로 구성)
<과정 정보 입력> 과정명, 과정기간, 강의실 정보 -> INSERT

관리자는 등록된 모든 과정의 정보를 출력
<출력 정보> 과정명, 강의실, 과목명, 과목기간, 교재명, 교수자명 -> VIEW

5. 관리자는 한 개 과정 당 여러 개의 과목을 미리 등록
   각 과목은 담당 교수자를 미리 배정 및 변경 가능
   -> INSERT, UPDATE -> TRIGGER (교수테이블 데이터 변경 시 -> 과목테이블도 변경)
<과목 정보 입력> 과정명, 과목명, 과목기간, 교재명, 교수자명 -> INSERT

각 항목의 배점은 담당 교수가 결정-> 총 배점은 100점 기준
INSERT -> PROCEDURE

관리는 등록된 모든 과목의 정보를 출력
<출력 정보> 과정명, 강의실, 과목명, 과목기간, 교재명, 교수자명 -> VIEW

6. 최종 관리자는 한 개 과정당 여러 명의 학생을 미리 등록
<학생 입력 정보> 학생명, 주민번호, 패스워드 -> INSERT

관리자는 등록된 모든 학생의 정보를 출력
<출력 정보> 학생명, 과정명, 수강과목, 수강과목 총점 -> VIEW
IF 중도탈락한 경우 중도탈락 사실 출력

7. 여러 개의 과목을 수강한 학생은 모든 과목을 한 페이지에서 확인할 수 있어야함
중도 탈락 되었더라도 이미 수강한 과목은 정보 출력 되어야함 -> VIEW
